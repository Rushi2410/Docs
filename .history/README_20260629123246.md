# 📄 Docs

A **production-grade Local-First Collaborative Document Editor** built using **Next.js 16**, **React 19**, **TypeScript**, **PostgreSQL**, **Prisma**, **TipTap**, **Yjs**, **Dexie**, and **Socket.IO**.

This project was developed for the **House of Edtech Full Stack Developer Assignment** and demonstrates modern full-stack engineering concepts including **offline-first architecture**, **real-time collaboration**, **deterministic conflict resolution**, **granular version history**, **role-based authorization**, **secure synchronization**, and **AI-powered writing assistance**.

# 💾 Local-First Design

The application is designed so the browser remains the primary working copy while editing.

Workflow:

1. User edits the document.
2. Every change is immediately applied to Yjs.
3. The latest state is stored in IndexedDB.
4. Incremental updates are queued locally.
5. The synchronization engine pushes queued updates when connectivity returns.
6. The server merges incoming updates safely.
7. The latest document state is returned to every connected client.

This architecture ensures the editor remains fully responsive even when the network is unavailable.

# 🛡 Security

Security measures include:

* JWT Authentication
* HttpOnly Cookies
* Password Hashing
* Zod Validation
* Prisma Membership Scoping
* Payload Size Validation
* Oversized Payload Rejection
* Viewer Write Protection

Malformed synchronization payloads are rejected before decoding to reduce Out-of-Memory (OOM) risks.

# 🚀 Getting Started

## Install Dependencies

```bash
npm install
```

## Configure Environment Variables

Create a `.env.local` file.

```env
DATABASE_URL="postgresql://..."
JWT_SECRET="your-secret-key"
NEXT_PUBLIC_APP_URL="http://localhost:3000"
GEMINI_API_KEY=""
```

## Generate Prisma Client

```bash
npm run prisma:generate
```

## Run Migrations

```bash
npm run prisma:migrate:dev
```

## Start Development Server

```bash
npm run dev
```

# 🚀 Deployment

The application requires:

* PostgreSQL Database
* Environment Variables
* Prisma Migrations

Deployment commands:

```bash
npm run prisma:migrate:deploy
npm run build
npm run start
```

> **Note:** This project uses a custom Node.js server (`server.js`) with Socket.IO. For full realtime collaboration, deploy to a Node.js-compatible platform such as **Render**, **Railway**, or **Fly.io**. The provided Vercel deployment demonstrates the application, but persistent Socket.IO functionality may require a traditional Node server.
